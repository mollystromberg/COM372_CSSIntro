Intro to CSS

An introduction and cheatsheet

CSS Rules!

CSS rules! because we can use it to set the visual style of our HTML documents, and create unique visual presentations that convey character while communicating visual hierarchy. We do this by defining CSS rules that tell the browser how to display our HTML content. A document will usually include many CSS rules, each made up of a selector followed by a list of properties and values. The selector tells the browser which page elements we want to apply a particular visual style to. The properties and values then define the visual elements (colors, fonts, sizes and more!) that make up the style. The format of the rule is always as follows:
selector {
property1: value1;
property2: value2;
}
 (Note: line breaks are optional, but recommended).

Selectors: tag, id, and class

We can select HTML elements to style in a several ways, by using different formats for our selector. We can select a tag by using the tag itself as the selector. For example:
h1 {
color: red;
}
 will make all h1 tags red. We can select an element by id by using the # character and then the id we want to select. For example:
#mainhead {
color: red;
}
 will make the element with the id "mainhead" red. We can select an element by class by using the . character and then the class we want to select. For example:
.bodypara {
color: red;
}
 will make all the elements with the class "bodypara" red.

Selector Cheat Sheet

  • tag - selects all instances of the given tag (ex: h1)
  • #id - selects all instances of the given id (ex: #mainhead)
  • .class - selects all instances of the given class (ex: .bodypara)

Cascading and Inheritance

When deciding how to display our HTML documents, our browser looks at several sources of css rules. These include:

  • The browser's default stylesheet
  • External stylesheet files included in the HTML with the link element
  • The style element of the HTML document

As long as these rules don't conflict, the browser will apply them all to the selected element. In the case of a conflict (for example, two different color settings for the same element) the browser will resolve the conflict as follows:

  • Included stylesheets over-ride the default style
  • Rules listed in later stylesheets over-ride earlier stylesheets
  • The style section over-rides external stylesheets

Elements also inherit some rules from their ancestors. If a rule is set that conflicts with the inherited rule, the inherted rule is over-rided.
